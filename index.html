Neden Docker?

✓ İzolasyon - Uygulamalar birbirinden bağımsız çalışır
✓ Ölçeklenebilirlik - Kolayca çoğaltılabilir
✓ Versiyon Kontrolü - Image'ler sürümlenebilir
✓ Kaynak Verimliliği - Daha az kaynak kullanır

Temel Docker Komutları

docker build -t myapp . Dockerfile'dan image oluştur
docker run -p 80:80 myapp Konteyner başlat ve port yönlendir
docker ps Çalışan konteynerleri listele
docker stop container_id Konteyneri durdur

Örnek Dockerfile

# Temel image
FROM nginx:alpine

# Çalışma dizini
WORKDIR /usr/share/nginx/html

# Dosyaları kopyala
COPY . .

# Port aç
EXPOSE 80

# Nginx başlat
CMD ["nginx", "-g", "daemon off;"]
